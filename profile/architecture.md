## 1. プロジェクト概要

### 1-1. 背景

介護現場では、以下の課題が存在している。

- 利用者の状態判断・リスク察知が属人化しており、
    
    必要な情報がその場で得られないことで **事故・ヒヤリハットにつながる**。
    
- 申し送りや情報共有が「インカム」「口頭」「紙」「バラバラなメモ」に分散しており、
    
    **新人が状況を把握しづらい**。
    
- ベテランであっても、忙しさや疲労により **業務の漏れや判断ミスが起こり得る**。
    
- 利用者を危険視しすぎる結果、必要以上に行動を制限してしまうケースもある。
    

これらの課題は本来の目的である「質の高い介護」を阻害している。

### 1-2. コアバリュー

> 介護の質を情報でサポートする

- 新人・ベテラン問わず、「その場で必要な情報」を提供し、
    
    **判断と行動の質を情報で底上げする**。
    
- AR デバイスと音声認識／要約／翻訳を活用し、
    
    **インカム・申し送り・個人情報をシームレスに統合する**。
    
- 情報がきちんと共有されることで、
    
    利用者の行動を以前より **予測しやすく**なり、
    
    **不必要な行動制限を減らす**ことを目指す。
    

### 1-3. 目的

- AR グラスを用いて、介護スタッフが **「今、その場で必要な情報」** にアクセスできるようにする。
    
- インカム音声と阿波弁 AI を活用し、
    
    - 音声を文字起こし・標準語要約・翻訳
    - タイムライン化していつでも振り返り可能にする
- 固定的な利用者情報（性格、訴えかけ方、なだめ方、持病、傾向）と
    
    流動的な情報（バイタル・申し送り・様子）を一元管理し、
    
    **「今日気をつけるべきポイント」** をスタッフに提示する。
    

### 1-4. 対象施設

- 特別養護老人ホーム（特養）を主な想定ターゲットとする。

## 2. ステークホルダー

### 2-1. 利用者

- 特別養護老人ホームに入居している利用者本人
    - 本システムの直接の利用者ではないが、
        
        **情報共有の質向上により安全性と QOL の向上が期待される。**
        

### 2-2. 介護スタッフ

- 新人介護職員
    - 申し送りや過去の様子を一度に把握しづらく、不安や負担が大きい。
- 中堅〜ベテラン介護職員
    - 自身の経験と勘で現場をまわしているが、
        
        情報の属人化・伝達漏れが起こり得る。
        

### 2-3. 管理者・看護師等

- 施設長・フロアリーダー・看護スタッフなど
    - **情報の記録・参照・共有状況**を俯瞰し、
        
        リスクや傾向を把握する必要がある。
        

### 2-4. 開発チーム

- バックエンド: TypeScript（Node / Lambda / Supabase など）
- フロントエンド: Flutter（主に Web ターゲット）
- デバイス: Raspberry Pi + Python, AR グラス用クライアント

## 3. スコープ

### 3-1. 機能要件（概要）

### 3-1-1. 個人情報管理

**固定されたデータ（スプレッドシート管理）**

- 持病

> 固定情報は主に PC から編集可能とし、
> 
> Google スプレッドシートを **マスタデータの編集 UI 兼ストレージ** として利用する。

**流動的なデータ**

- バイタル（体温、血圧 等）
    - MVP：スプレッドシートに手入力（既存業務フローを想定）
    - 将来：計測機器・他システムとの連携も視野に入れる（非必須）
- 申し送り（要約テキスト）
    - 元の音声はクラウドストレージ（AWS S3 or Supabase Storage）に保存
    - Whisper + 阿波弁 AI による標準語テキスト要約を生成
    - 要約テキストはスプレッドシートにも保存し、スタッフが編集可能とする。
- 最近の様子
    - 特筆すべき変化や様子を、
        
        **LLM API が自動的に抽出・テキストとして記録**する構成を想定
        

### 3-1-2. 情報共有機能（インカム・タイムライン）

- インカム・ピッチによるスタッフ間会話を前提とし、それを上位互換にすることを目指す。
- 音声ログを収集し、以下の処理を行う：
    - Whisper による音声認識（阿波弁対応）
    - 阿波弁 AI による標準語正規化・翻訳
    - 話者識別（誰が話したか B 案）
    - 要約・重要ポイント抽出
- 出力：
    - タイムラインイベントとして保存（DB）
    - 「今日気をつけるポイント」として集約（スプレッドシート & UI）

### 3-1-3. タイムライン表示・閲覧

- 介護スタッフが、**インカム発話や申し送り・イベントの履歴**をタイムラインとして閲覧できる。
- 単位：
    - 日別
    - 利用者別
    - フロア／シフト単位など（将来的拡張）
- フロントエンド：
    - AR グラス：直近の要約・注意ポイントをポップアップ表示
    - Web ダッシュボード（PC / タブレット）：詳細な履歴・フィルタ閲覧
    - スマホ対応は将来検討（MVP では必須としない）

### 3-1-4. 「今日気をつけるべきポイント」の自動生成

- シフトイン時に、担当利用者ごとに以下をまとめた情報を提示する：
    - 前日〜直近の申し送り要約
    - バイタルの異常傾向（あれば）
    - 行動変化（様子データからの抽出）
    - 最後に排泄した時間／睡眠状況 等（取得できる範囲で）
- 生成された「今日気をつけるべきポイント」は：
    - スプレッドシートに保存
    - Web ダッシュボードおよび AR ポップアップから参照

### 3-2. 非機能要件（範囲）

- 本システムは **正式な電子カルテ・電子記録システムの代替を目的としない**。
    - 法令で定められた記録保存義務・監査要件などの「法的お墨付き」は要求しない。
- 目的はあくまで
    - **介護現場の円滑な業務進行**
        
    - 情報共有の質向上
        
    - ヒヤリハット・事故の抑制
        
        をサポートすることとする。

## 4. ゴール

- 1つの業務（1件のケア・対応）にかかる **情報探索・確認時間を削減**する。
    
- 記録（音声・テキスト・バイタル）に **追加の価値（要約・傾向・ポイント抽出）** を付与する。
    
- 新人・ベテラン間での情報ギャップを小さくし、
    
    **質の高いコミュニケーション**と**質の高い介護**につなげる。
    
- 利用者の安全性を保ちながら、
    
    **不必要な行動制限を減らすことができる状態**を目指す。

## 5. ユースケース

### 5-1. ユースケース一覧（例）

1. **シフトイン時の情報把握**
    - 介護職員がシフトに入る際、担当利用者ごとの「今日気をつけるべきポイント」を確認する。
2. **申し送り音声の自動記録・要約**
    - インカムでの会話内容が自動で文字起こし・要約され、タイムラインに記録される。
3. **AR でのその場情報参照**
    - 利用者の前に立つと顔認識などをトリガーに、その利用者の固定情報＋今日の注意点を AR にポップアップ表示。
4. **PC ダッシュボードでの後追い確認**
    - 日中やシフト後に PC で、申し送り履歴・タイムライン・様子をまとめて振り返る。

### 5-2. 代表シナリオ

### シナリオ：シフトインした介護職員が「今日気をつけるべきポイント」を確認する

1. 職員がシフトインし、担当利用者一覧を確認する。
    
2. システムは以下の情報をもとに、担当者ごとの「今日気をつけるべきポイント」を生成・表示する：
    
    - 前日〜今朝までの申し送り要約
    - 直近のバイタル情報（異常値・注意すべき変化）
    - 様子（LLMによる特記事項の自動抽出）
    - 最後の排泄、睡眠、行動の変化等（取得可能な範囲）
3. 職員は AR グラス / Web ダッシュボードでこれらを確認し、
    
    その日一日のケアのイメージ・注意すべきポイントをつかむ。
    
4. 日中に新たな出来事があれば、インカムでの会話が音声として記録され、
    
    AI により必要な部分がタイムライン・申し送り要約として反映される。
